# FormaUI v0.8 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 基于 `docs/prds/FormaUI v0.8 PRD.md` 完成 FormaUI 文档服务升级，采用 Fumadocs 原生架构，统一 `/docs/*` 路径，并将文档结构对齐主流 UI 框架的 IA 设计。

**Architecture:** 保持 monorepo 不变，在 `apps/docs` 内完成文档内核替换：以 `content/docs` 为唯一内容源、Fumadocs `source` 驱动页面渲染与自动导航，逐步迁移现有 MDX 页面并建立搜索、SEO 与发布治理能力。执行上采用“先基础设施、后核心栏目、再全量迁移与硬化”的三段式交付。

**Tech Stack:** Next.js 15, React 19, TypeScript, Fumadocs (`fumadocs-ui`, `fumadocs-core`, `fumadocs-mdx`), Tailwind CSS, Vitest, pnpm workspace, Turborepo.

- 每个内容域迁移必须满足“三件套”：页面可访问 + 导航可见 + 搜索可命中（核心关键词抽样）。

## 4. Cross-Version Engineering Rules

- 以 PRD 为唯一范围基线；不扩展到 CMS 重构、营销站 IA 重构、多语言正式上线。
- Fumadocs 优先采用官方推荐实现，避免无必要自定义渲染管线。
- `content/docs` 是唯一文档事实源；禁止继续新增 `app/**/page.mdx` 作为长期入口。
- 导航一律通过 page tree + `meta.json` 管理，不再维护手工长数组导航。
- 每次迁移批次必须更新对应测试，避免“内容迁过去但导航/搜索失效”。
- 保持 docs 子应用可独立验证：
  - `pnpm --filter docs lint`
  - `pnpm --filter docs test`
  - `pnpm --filter docs build`

## 13. Risks and Mitigations

1. 风险：一次性切换路径引发内部/外部链接失效。  
缓解：在 v0.8 release notes 和 guides 明确新路径规则，并提供入口页导流。

2. 风险：批量迁移导致页面文件存在但导航不可见。  
缓解：以 `meta.json` 为中心建立 IA 测试，禁止仅凭人工点查。

3. 风险：Fumadocs 接入后视觉差异过大影响体验。  
缓解：v0.8 先保结构与能力一致，视觉细化留在后续子版本。

4. 风险：搜索初期召回不足。  
缓解：建立关键词抽样表（组件名、CLI 命令、迁移词）做每版回归。
