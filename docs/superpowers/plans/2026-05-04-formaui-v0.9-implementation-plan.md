# FormaUI v0.9 Landing Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade `apps/web` into a polished, conversion-focused FormaUI landing page based on `docs/prds/FormaUI v0.9 PRD.md`.

**Architecture:** Keep the existing Next.js App Router web app and section-based homepage architecture. Replace the current generic SaaS marketing sections with a focused landing narrative: source-owned UI, real product surfaces, registry/CLI build flow, scenario gallery, quality proof, and clear Docs/GitHub conversion paths. Use server components by default; only the product surface switcher may use a client component.

**Tech Stack:** Next.js 15, React 19, TypeScript, Tailwind CSS 3, Vitest, `react-dom/server`, pnpm workspace, Turborepo.
