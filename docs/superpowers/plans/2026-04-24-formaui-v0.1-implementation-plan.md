# FormaUI v0.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build FormaUI v0.1 as a source-owned design system that can initialize a React project, copy components and product blocks from a registry, and document the workflow end to end.

**Architecture:** Use a pnpm monorepo with separated packages for tokens, themes, components, blocks, registry generation, CLI, Tailwind preset, docs, and example app. Source files live in packages, registry items describe installable assets, and the CLI copies resolved files into target projects. Docs and the Next.js example app validate that the source-owned workflow is understandable and runnable.

**Tech Stack:** React, TypeScript, Tailwind CSS, CSS Variables, Radix UI, class-variance-authority, lucide-react, pnpm workspace, Turborepo, tsup, Vitest, Playwright, Next.js.

---

## 1. Version Breakdown

v0.1 is split into six executable subversions:

| Version | Name | Primary Outcome |
| --- | --- | --- |
| v0.1.0 | Monorepo Foundation | Repo scaffold, package boundaries, build/test/lint baseline |
| v0.1.1 | Design Foundation | Tokens, themes, dark mode, Tailwind preset |
| v0.1.2 | Core Components | 13 primitive components with demos and tests |
| v0.1.3 | Registry & CLI Alpha | Registry schema, generated registry items, `init`, `add`, conflict handling |
| v0.1.4 | Product Blocks & Template | 8 required blocks, 8 composite components, `ai-console-lite` template |
| v0.1.5 | Docs, Example & Release Hardening | Docs site, Next.js example, complete verification, v0.1 release notes |

Each subversion must leave the repository in a runnable state. Do not start the next subversion until the current subversion's exit criteria pass.

## 3. Cross-Version Engineering Rules

- Use TypeScript strict mode.
- Keep source-owned install targets free of hidden runtime dependencies on private FormaUI packages.
- Every installable asset must have a registry item before being documented as installable.
- Prefer small component files with colocated tests and demos.
- Commit at the end of every task if the repository is a git repository.
- Run the verification commands listed in each task before marking it complete.
- Do not document commands that are not implemented.
