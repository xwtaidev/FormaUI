# FormaUI v0.5 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 交付 v0.5 双站拆分（`apps/web` + `apps/docs`），实现“品牌官网”和“官方文档”职责解耦，并完成跨站导航、旧路由迁移与发布级验证。

**Architecture:** 保持 monorepo + Turborepo 架构不变，在 `apps/*` 下新增独立 Next.js Web 应用承载品牌与增长页面；`apps/docs` 收敛为纯文档站，仅保留安装、组件、Blocks、Templates、CLI、Registry、Migration 文档。通过跨站入口与重定向策略保证迁移期间无关键路径断裂。

**Tech Stack:** Next.js (App Router), React, TypeScript, MDX（docs）、Tailwind CSS、pnpm workspace、Turborepo、Vitest。

## 3. Cross-Version Engineering Rules

- `apps/web` 仅承载品牌官网相关页面：首页、营销页、产品介绍、场景展示、Showcase、Blog、Changelog 宣传页。
- `apps/docs` 仅承载文档体系：安装、组件、Blocks、Templates、CLI、Registry、Migration Guides。
- 双站必须互链：`Web -> Docs`、`Docs -> Web` 在首屏可见位置可达。
- 重定向策略必须可测试：关键旧路由不能出现未预期 404。
- 文档与官网都要有独立 metadata、sitemap、robots 输出。
- `apps/web/app/page.tsx` 只允许做区块编排，不允许写数据请求和复杂业务逻辑。
- Web 首页区块组件统一放在 `apps/web/app/_sections/`，文件命名统一为 `landing-*.tsx`。
- 首页文案和统计数据集中在 `apps/web/app/_sections/content.ts`，区块组件只消费数据。
- 必须有“首页区块顺序测试”，断言区块顺序不被破坏。

Web 首页编排契约（强约束）：

```tsx
export default function LandingPage() {
  return (
    <main>
      <LandingHeader />
      <LandingHero />
      <LandingStats />
      <LandingFeatures />
      <LandingProof />
      <LandingScenarios />
      <LandingQuickStart />
      <LandingBlocksFlow />
      <LandingQualityGates />
      <LandingFaq />
      <LandingFinalCta />
      <LandingFooter />
      <LandingAnalytics />
    </main>
  );
}
```

通用回归命令：

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm build
```

Expected:

```txt
All commands exit with code 0.
Both apps can start/build independently.
```
