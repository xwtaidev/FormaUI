# FormaUI v0.3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver FormaUI v0.3 with a larger installable component catalog and scenario-based pack integration while preserving v0.2 workflow reliability.

**Architecture:** Keep the existing source-owned flow (`registry -> CLI -> copied source files -> docs/examples`) and extend it in three layers: (1) Registry v3 metadata + pack type, (2) component waves with strict testing gates, (3) CLI/docs/example upgrades that make larger asset catalogs discoverable and installable. Release in six subversions so every checkpoint remains runnable and releasable.

**Tech Stack:** React, TypeScript, Tailwind CSS, Radix UI, pnpm workspace, Turborepo, Vitest, Next.js, Vite.

## 3. Cross-Version Engineering Rules

- TDD first for CLI/registry/components (write failing tests before implementation).
- Keep exported component APIs minimal and predictable; avoid hidden runtime dependencies.
- Update component barrel exports and registry items in the same task to prevent drift.
- Every task ends with focused commit; no mixed-scope commits.
- Release verification gate after each subversion:

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm build
pnpm registry:build
```

Expected:

```txt
All commands exit with code 0.
```
