# FormaUI v0.6 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 交付 v0.6「基础组件补齐版」，新增 20 个基础组件并完成源码、registry、docs、测试、示例与发布级验证闭环。

**Architecture:** 保持现有 monorepo 架构不变，在 `@formaui/components` 扩充 primitives；通过 `@formaui/registry` 生成可安装条目；在 `apps/docs` 补齐组件文档与导航可发现性；在 `examples/next-app` 进行集成展示与回归验证。

**Tech Stack:** React 19, TypeScript, Radix UI, Tailwind CSS, Vitest, Next.js (docs/examples), pnpm workspace, Turborepo.

## 4. Cross-Version Engineering Rules

- 命名统一按 PRD P0 清单执行，不在 v0.6 引入双名官方组件（例如仅交付 `drawer`，不新增 `sheet` 作为独立条目）。
- 组件实现优先复用既有设计 token、`cn`、`class-variance-authority` 与现有 API 约定（`variant`/`size`/`asChild`）。
- Radix-first 组件进入 `primitives-radix.test.tsx` 覆盖；非 Radix 组件进入 `primitives-basic.test.tsx` 覆盖。
- 所有组件必须导出并可通过 `@formaui/components` 单包入口消费。
- `packages/registry/src/items/components.ts` 是 registry 单一事实来源，禁止手写绕过流程生成 JSON。
- 文档导航可发现性是硬性要求，`apps/docs/app/layout.tsx` 必须追加 20 个组件条目。
- 每波结束运行：
  - `pnpm --filter @formaui/components test`
  - `pnpm registry:build`
  - `pnpm --filter docs build`
