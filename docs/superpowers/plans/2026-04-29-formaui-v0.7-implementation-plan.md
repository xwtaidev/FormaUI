# FormaUI v0.7 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 交付 v0.7「基础组件完备版」，新增 16 个基础组件，并完成源码、registry、docs、测试、示例与发布级验证闭环。

**Architecture:** 保持当前 monorepo 架构不变，在 `@formaui/components` 继续扩充 primitives；通过 `@formaui/registry` 生成可安装条目；在 `apps/docs` 补齐组件文档与导航可发现性；在 `examples/next-app` 提供 v0.7 组合示例并做回归验证。

**Tech Stack:** React 19, TypeScript, Radix UI, Tailwind CSS, Vitest, Next.js (docs/examples), pnpm workspace, Turborepo.

## 4. Cross-Version Engineering Rules

- 严格按 v0.7 PRD 的 P0 16 组件范围执行，不混入 P1 组件（如 `carousel`、`statistic`、`mentions`）。
- 新组件实现优先复用既有 token/theme、`cn`、`class-variance-authority` 与已存在 primitives API 风格。
- 能用组合实现的能力优先组合（如 `time-picker` 可组合 `input/popover`），避免过重实现。
- Radix 依赖仅在必要时引入，新增依赖需在 PR 里写明必要性。
- 所有组件必须导出并可通过 `@formaui/components` 统一入口消费。
- `packages/registry/src/items/components.ts` 仍是 registry 单一事实来源，禁止手写绕过流程 JSON。
- 文档导航可发现性是硬约束，`apps/docs/app/layout.tsx` 必须新增 16 个组件入口。
- 每波结束最低验证：
  - `pnpm --filter @formaui/components test`
  - `pnpm registry:build`
  - `pnpm --filter docs test && pnpm --filter docs build`
