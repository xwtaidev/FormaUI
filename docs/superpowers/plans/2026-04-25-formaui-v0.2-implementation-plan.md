# FormaUI v0.2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver FormaUI v0.2 as a reliable, scalable source-owned design system release across CLI, registry, assets, docs, and examples.

**Architecture:** Build v0.2 in six release-safe subversions (`v0.2.1` to `v0.2.6`) that each leave the repo runnable. Prioritize workflow reliability first (CLI + registry), then expand assets (components/blocks/templates), then finish docs/examples and release hardening. Keep all installable assets registry-driven and source-owned.

**Tech Stack:** React, TypeScript, Tailwind CSS, Radix UI, pnpm workspace, Turborepo, Next.js, Vite, Vitest.

---

| Version | Name | Primary Outcome |

| v0.2.6 | Docs/Examples/Release Hardening | Full docs coverage, `examples/vite-app`, release checklist completion |

Rules:

- Do not start the next subversion until current exit criteria pass.
- Every subversion must keep `pnpm lint && pnpm typecheck && pnpm test && pnpm build` green.
- Every shipped installable asset must have a registry item before docs/examples mention it.

## 3. Cross-Version Engineering Rules

- Keep TypeScript strict mode on for all new files.
- Preserve source-owned delivery: copied files must not depend on hidden private runtime code.
- Use TDD for CLI, registry, and render-critical assets.
- Prefer small files with focused responsibilities.
- Commit at the end of each task with explicit message scope.
- Never document a CLI command before implementation + tests land.

Baseline verification command for every subversion:

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm build
```

Expected result:

```txt
All commands exit with code 0.
```
