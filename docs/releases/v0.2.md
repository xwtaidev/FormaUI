# FormaUI v0.2 Release Notes

Date: 2026-04-25  
Status: GA  
Version Line: v0.2.1 -> v0.2.6

## TL;DR

FormaUI v0.2 将 v0.1 的“可跑通”升级为“可规模使用”，重点完成四件事：

- CLI 从基础安装工具升级为可发现、可诊断、可预演（dry-run）的工作流入口。
- Registry 升级为 v2：支持版本化元数据、索引清单、远程源+本地回退。
- 新增一批可安装资产（组件、Blocks、模板），并补齐主题语义 token 适配。
- 文档与示例覆盖完成，形成 Next.js + Vite 双官方示例。

## Delivered Scope

### 1. CLI v2

新增命令：

- `npx formaui list [components|blocks|templates|themes]`
- `npx formaui search <keyword>`
- `npx formaui info <name>`
- `npx formaui doctor`

新增能力：

- `--dry-run`：在不落盘前提下输出文件写入计划和依赖变更计划。
- `--registry <url|path>`：支持远程 registry 与本地路径切换。
- `init/add/block add/template add/theme add` 可靠性增强：依赖安装、冲突/失败回滚、诊断提示更完整。

### 2. Registry v2

Schema 元数据扩展字段：

- `version`
- `description`
- `tags`
- `frameworks`
- `sources`
- `checksum`（optional）

交付内容：

- `registry/index.json` 版本化索引（支持 kind/name/version 维度查询）。
- CLI 远程读取 registry 失败时，本地回退路径可用。

### 3. Assets

新增组件：

- `data-table`
- `search-command`

新增 Blocks：

- `team-members-table`
- `notification-panel`
- `model-selector`

新增模板：

- `saas-starter`
- `admin-dashboard`

主题与 token：

- 维持 `default`、`avocado` 双主题。
- 新资产兼容语义 token 分层。

### 4. Docs and Examples

- 可安装资产文档覆盖完成（components/blocks/templates/themes）。
- 迁移文档：`apps/docs/app/migration-v0.1-to-v0.2/page.mdx`
- CLI 命令参考：`apps/docs/app/cli/commands/page.mdx`
- 官方示例：`examples/next-app` + `examples/vite-app`

## Upgrade Notes (v0.1 -> v0.2)

### Behavior Changes

- `add` 系列命令会自动安装依赖（按项目包管理器执行）。
- `init` 会尝试注入 `styles/formaui.css` 引用并记录 CSS entry。
- 建议在现有项目先运行 `doctor` 和 `--dry-run` 再正式安装资产。

### Recommended Migration Steps

1. 使用 v0.2 CLI 在项目根目录执行 `npx formaui doctor`。
2. 用 `--dry-run` 预演目标安装动作（例如 `npx formaui add button --dry-run`）。
3. 确认计划输出后执行真实安装命令。
4. 运行项目校验：`pnpm lint && pnpm typecheck && pnpm test && pnpm build`。

## Validation Gates

Release gating commands:

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm build
```

Release condition:

- All commands exit with code `0`.

## Compatibility

- Framework: React (officially supported in v0.2)
- Tooling: Next.js, Vite (React + TS official examples)
- Package managers: `pnpm`, `npm`, `yarn`, `bun` (CLI install flow)

## Known Limitations

- Official framework support remains React-focused.
- Private registry auth workflows are not included in v0.2.
- Figma kit / design token bidirectional sync is out of scope for v0.2.
