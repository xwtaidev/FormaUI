# FormaUI v0.2 Release Notes

Date: 2026-04-25  
Status: GA

## Summary

FormaUI v0.2 upgrades v0.1's validated source-owned workflow into a release-ready toolkit across CLI, registry, assets, docs, and examples.

## Delivered Scope

### CLI v2

- Added `list`, `search`, `info`, `doctor`.
- Added `--dry-run` planning mode for install commands.
- Added stronger install reliability paths for `init/add/block/template/theme`.

### Registry v2

- Added metadata fields: `version`, `description`, `tags`, `frameworks`, `sources`, `checksum`.
- Added generated `registry/index.json` for versioned item discovery.
- Added remote registry resolution with local fallback behavior.

### Assets

- Components: `data-table`, `search-command`.
- Blocks: `team-members-table`, `notification-panel`, `model-selector`.
- Templates: `saas-starter`, `admin-dashboard`.
- Themes: `default`, `avocado` with semantic-token compatibility.

### Docs and Examples

- Completed docs coverage for all installable components, blocks, templates, and themes.
- Added migration guide: `v0.1 -> v0.2`.
- Added CLI command reference page.
- Added official `examples/vite-app` alongside `examples/next-app`.

## Verification Gates

The following release gates were executed for v0.2 final validation:

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm build
```

Expected release condition: all commands exit with code `0`.

## Known Limitations

- Official framework support remains React-focused.
- Private registry auth workflows are not part of v0.2.
- Visual studio / Figma kit bidirectional tooling remains out of scope for this release.
