# FormaUI v0.3 Release Notes

Date: 2026-04-25  
Status: GA  
Version Line: v0.3.1 -> v0.3.6

## TL;DR

FormaUI v0.3 将 v0.2 的“可规模使用”升级为“可组合扩展”，核心交付如下：

- 新增 12 个可安装组件，总组件数达到 36。
- 引入 3 个官方 pack，并支持 `pack list/info/add` 场景化安装。
- Registry 升级到 v3 元数据能力，同时保持对 v0.2 资产的兼容读取。
- 文档与 Next/Vite 示例完成 v0.3 覆盖，并通过完整发布门禁。

## Delivered Scope

### 1. Component Catalog Expansion

v0.3 新增组件（12）：

- Primitives: `accordion`, `popover`, `hover-card`, `progress`, `separator`, `skeleton`, `radio-group`
- Composites: `date-range-picker`, `filter-bar`, `data-table-toolbar`, `pagination-bar`, `empty-search-state`

结果：

- 安装资产总量达到 36 个组件。
- 每个新增组件都具备 registry item、文档页、示例接入和测试覆盖。

### 2. Pack Workflow

新增官方 pack：

- `dashboard-foundation`
- `data-entry`
- `feedback-loading`

CLI 新增命令：

```bash
npx formaui pack list
npx formaui pack info <name>
npx formaui pack add <name>
```

### 3. Registry v3 Discoverability

Registry v3 新增结构化字段：

- `category`
- `scenarios`
- `complexity`
- `stability`

CLI 支持基于 v3 元数据过滤：

```bash
npx formaui list components --category form
npx formaui search table --scenario dashboard
```

### 4. Docs and Official Examples

- 文档新增 12 个组件页与 pack 专题页。
- 完成 `v0.2 -> v0.3` 迁移文档。
- `examples/next-app` 与 `examples/vite-app` 均覆盖 v0.3 资产并通过构建验证。

## Migration Notes (v0.2 -> v0.3)

### New Recommended Flow

1. 先运行 `npx formaui doctor` 检查当前项目状态。
2. 使用 `list/search` + metadata filter 选择组件或 pack。
3. 优先使用 `pack add` 完成场景化批量安装，再补充单组件安装。
4. 安装后执行项目校验：`pnpm lint && pnpm typecheck && pnpm test && pnpm build`。

### Command Additions in v0.3

- `npx formaui pack list`
- `npx formaui pack info <name>`
- `npx formaui pack add <name>`
- `--category` / `--scenario` discoverability filters

## Validation Gates

Release gating commands:

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm build
pnpm registry:build
```

Release condition:

- All commands exit with code `0`.

Validation evidence snapshot:

- Registry build output: `36 component items`, `2 theme items`, `11 block items`, `3 template items`, `3 pack items`, `55 index entries`.
- Full workspace gates were executed during v0.3.6 closure with zero failing tasks.

## Upgrade Checklist (v0.2 -> v0.3)

1. Run `npx formaui doctor` in project root.
2. Use `npx formaui pack list` / `npx formaui pack info <name>` to choose scenario packs.
3. Dry-run first for high-impact installs: `npx formaui pack add <name> --dry-run`.
4. Execute real install and then run project checks.
5. Validate `pnpm lint && pnpm typecheck && pnpm test && pnpm build`.

## Compatibility

- Framework: React (official support line)
- Tooling: Next.js, Vite (official examples)
- Package managers: `pnpm`, `npm`, `yarn`, `bun`
- Registry compatibility: v0.2 items and v0.3 items can be parsed by current CLI flow

## Known Limitations

- 官方框架支持仍以 React 为主，跨框架生态暂未纳入 v0.3 正式范围。
- `recommend` 智能推荐能力仍处于后续规划，不在 v0.3 GA 范围内。
- 私有 registry 鉴权/权限体系仍未纳入当前版本。
- Vite build may print third-party `"use client"` directive warnings during bundling; this is currently non-blocking and does not fail release gates.
