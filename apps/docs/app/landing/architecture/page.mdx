# Landing Architecture (v0.4.4)

## Scope

The official landing route lives at `/landing` in `apps/docs` and is composed from section-level containers:

- `landing-hero.tsx`
- `landing-proof.tsx`
- `landing-cta.tsx`
- `content.ts` (copy and structured section data)

## Information Architecture

Landing section order is fixed and validated by tests:

1. `hero`
2. `core-differentiators`
3. `asset-proof`
4. `scenario-showcase`
5. `faq`
6. `quick-start-path`
7. `landing-footer`

## Component Mapping

Each section must be driven by FormaUI assets:

- Hero: `HeroCta`
- Differentiators: `FeatureGrid`
- Proof: `StatsStrip`
- Showcase: `LogoCloud`
- FAQ: `FaqAccordion`
- Final path: `FinalCta`

## Import Policy

Landing files under `apps/docs/app/landing/**/*.tsx` are checked by:

```bash
pnpm --filter docs lint:landing-components
```

Allowed import families:

- `@formaui/components`
- `@formaui/blocks`
- `react`, `next/*`, `node:*`
- local relative files (`./`, `../`) and `@/` aliases

## Contributor Constraints

Do:

- Keep section composition in container components.
- Keep copy updates in `content.ts`.
- Prefer adding shared UI to FormaUI packages before use in landing.

Don't:

- Import third-party UI packages directly into landing route files.
- Duplicate large JSX blocks inside `page.tsx`.
- Introduce ad-hoc primitives that bypass the shared design system.
