# CLI

FormaUI v0.2 CLI focuses on reliable install flows, discoverability, and diagnostics.

## Core Commands

```bash
formaui init
formaui add <name>
formaui block add <name>
formaui template add <name>
formaui theme add <name>
formaui list [components|blocks|templates|themes]
formaui search <keyword>
formaui info <name>
formaui doctor
```

## Shared Flags

- `--cwd <path>`: run command against another project path.
- `--yes`: non-interactive overwrite confirmation.
- `--registry <url|path>`: use remote or local registry source.
- `--dry-run`: print install plan only, no file writes.

## `formaui init`

Initializes baseline configuration and utility files.

Typical outputs:

- `formaui.json`
- `lib/utils.ts`
- `styles/formaui/default.css`

Requires: target project must have a `react` dependency.

## `formaui add <name>`

Installs a component registry item and all `registryDependencies` recursively.

## `formaui block add <name>`

Installs a block registry item and required dependencies.

## `formaui template add <name>`

Installs template entry files and dependencies.

## `formaui theme add <name>`

Installs a theme CSS registry item.

## Conflict Behavior

When target files already exist and `-y/--yes` is not provided, CLI exits with conflict details and does not write partial output.

## Diagnostics

Use `formaui doctor` to report project readiness:

- React dependency status
- Tailwind presence
- TypeScript and config checks
- CSS entry and FormaUI variable file status
