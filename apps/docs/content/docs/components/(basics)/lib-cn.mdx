# LibCn

`cn` 是 FormaUI 的基础工具函数，用于组合多个 className 输入，并自动处理 Tailwind 冲突类。

它内部等价于 `twMerge(clsx(...))`：先做条件拼接，再做冲突合并，保证最终输出可预测。


```bash title="Package Manager"

```

## 使用方式

```tsx title="Basic Usage"
import { cn } from "@formaui/components";

export function Demo({ active }: { active: boolean }) {
  return (
    <div
      className={cn(
        "rounded-md border px-3 py-2 text-sm",
        active ? "border-primary text-primary" : "border-border text-muted-foreground"
      )}
    >
      当前状态
    </div>
  );
}
```

## API Reference

### 函数签名

```ts title="Signature"
cn(...inputs: ClassValue[]): string
```

### 参数说明

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `inputs` | `ClassValue[]` | 支持字符串、数组、对象、布尔值、`null`、`undefined` 等 `clsx` 可解析输入 |

### 返回值

| 返回值 | 类型 | 说明 |
| --- | --- | --- |
| `className` | `string` | 拼接并冲突合并后的最终类名字符串 |

### 行为规则

- 先执行 `clsx`：处理条件逻辑与多形态输入。
- 再执行 `tailwind-merge`：同组冲突类保留“最后一个”。
- 对非冲突类保持原有顺序，避免不可预期样式抖动。

## 可访问性

- `cn` 本身不处理语义，仅负责类名计算。
- 建议将 `cn` 与语义化标签、`aria-*` 属性配合使用。
- 避免仅通过颜色变化表达状态，应搭配文本说明（如“已上线”、“处理中”）。
