# Typography

`Typography` 用于统一文本的语义与视觉层级，避免在业务页面中重复手写零散的字体样式。

它通过 `variant` 定义常见文本角色，并允许通过 `as` 覆盖渲染标签，兼顾可访问性与样式一致性。

## API Reference

### Props

| Prop | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `variant` | `"h1" \| "h2" \| "h3" \| "h4" \| "lead" \| "body" \| "small" \| "muted" \| "code"` | `"body"` | 文本样式变体 |
| `as` | `keyof React.JSX.IntrinsicElements` | 按 `variant` 自动推断 | 覆盖渲染标签（如 `h2` 样式但渲染成 `h3`） |
| `className` | `string` | `-` | 追加自定义样式 |

### Variant 语义映射

| variant | 默认标签 | 推荐用途 |
| --- | --- | --- |
| `h1` | `h1` | 页面主标题 |
| `h2` | `h2` | 章节标题 |
| `h3` | `h3` | 小节标题 |
| `h4` | `h4` | 组内标题 |
| `lead` | `p` | 导语/摘要 |
| `body` | `p` | 正文段落 |
| `small` | `small` | 标签、补充说明 |
| `muted` | `p` | 次要信息 |
| `code` | `code` | 行内代码片段 |

### 继承属性

`Typography` 继承 `React.HTMLAttributes<HTMLElement>`，可直接使用：

- `id`
- `aria-*`
- `data-*`
- `onClick`

## 可访问性

- 默认按语义标签渲染，利于屏幕阅读器理解文档结构。
- 当需要视觉与语义分离时，使用 `as` 覆盖标签并保持层级正确。
- 标题建议按 `h1 -> h2 -> h3` 顺序递进，避免跳级。
