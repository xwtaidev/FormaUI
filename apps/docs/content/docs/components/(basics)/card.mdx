# Card

`Card` 是用于承载一组相关信息的结构化容器，适合仪表盘、列表详情、设置面板等场景。

它通过 `CardHeader`、`CardTitle`、`CardDescription`、`CardContent`、`CardFooter` 组合出稳定的内容层级，而不是依赖单一大而全的 props。

## 安装方式

### 通过 CLI 安装

```bash title="CLI"
npx formaui add card
```

### 通过包依赖使用

```bash title="Package Manager"
pnpm add @formaui/components
```

## API Reference

### Card Props

`Card` 及其子组件均继承 `React.HTMLAttributes<HTMLDivElement>`（`CardTitle` 为 `React.HTMLAttributes<HTMLHeadingElement>`，`CardDescription` 为 `React.HTMLAttributes<HTMLParagraphElement>`）。

| 组件 | 默认样式职责 | 说明 |
| --- | --- | --- |
| `Card` | 外层容器、边框、背景、阴影 | 作为承载内容的根节点 |
| `CardHeader` | 顶部区块间距 | 放置标题和描述 |
| `CardTitle` | 标题字体层级 | 放置主要标题 |
| `CardDescription` | 次级文本样式 | 放置补充说明 |
| `CardContent` | 正文内容区间距 | 放置业务主体内容 |
| `CardFooter` | 底部操作区布局 | 放置按钮或辅助操作 |

### 常用属性

| Prop | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `className` | `string` | `-` | 追加自定义样式 |
| `children` | `ReactNode` | `-` | 渲染内容 |
| `id` | `string` | `-` | DOM 标识 |
| `aria-*` | `string` | `-` | 可访问性属性 |

## 可访问性

- 建议在卡片中使用清晰的标题层级（如 `CardTitle`）。
- 若卡片用于操作入口，建议在 `CardFooter` 中放置明确按钮文案。
- 对可点击整卡场景，需补充键盘可达与焦点样式，而非仅绑定鼠标事件。
