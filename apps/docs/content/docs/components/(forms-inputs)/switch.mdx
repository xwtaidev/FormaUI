# Switch

`Switch` 用于在两个互斥状态间快速切换，常见于通知配置、功能开关、偏好设置等场景。

它基于 Radix Switch 构建，提供清晰的启用/禁用反馈，并默认继承 FormaUI 主题与焦点样式。

```tsx title="components/demos/radio-switch-demos.tsx"
"use client";

import * as React from "react";
import { Switch } from "@formaui/components";

export function SwitchInteractiveDemo() {
  const [enabled, setEnabled] = React.useState(true);

  return (
    <label
      htmlFor="switch-demo-enabled"
      className="flex items-center justify-between gap-3"
    >
      <span className="text-sm">自动保存</span>
      <Switch
        id="switch-demo-enabled"
        checked={enabled}
        onCheckedChange={setEnabled}
      />
    </label>
  );
}
```

## API Reference

### Props

| Prop              | 类型                         | 默认值  | 说明           |
| ----------------- | ---------------------------- | ------- | -------------- |
| `checked`         | `boolean`                    | `-`     | 受控开关状态   |
| `defaultChecked`  | `boolean`                    | `false` | 非受控初始状态 |
| `onCheckedChange` | `(checked: boolean) => void` | `-`     | 状态变更回调   |
| `disabled`        | `boolean`                    | `false` | 禁用交互       |
| `required`        | `boolean`                    | `false` | 表单必填校验   |
| `name`            | `string`                     | `-`     | 表单字段名     |
| `value`           | `string`                     | `"on"`  | 表单提交值     |
| `id`              | `string`                     | `-`     | 用于关联 label |
| `className`       | `string`                     | `-`     | 追加自定义样式 |

### 继承属性

`Switch` 继承 `React.ComponentPropsWithoutRef<typeof SwitchPrimitive.Root>`，可直接使用：

- `aria-*`
- `data-*`
- `onFocus` / `onBlur`
- `onKeyDown`

## 可访问性

- 基于 Radix Switch，默认提供屏幕阅读器所需语义与键盘可操作性。
- 建议与可见文本标签配对，避免仅靠图标表达开关含义。
- 对于关键配置，建议同时提供状态说明文案（如“已开启/已关闭”）。
- 禁用状态应说明原因，避免用户误判为可操作异常。
