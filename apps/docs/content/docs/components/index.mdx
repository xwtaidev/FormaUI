# Components Overview

Components 是 FormaUI 的核心能力层，提供可复用的基础组件与组合组件。

本章节按工程实践拆分为多个文件夹分组，目标是让你在真实业务里更快完成“选型、接入、定制、验收”的闭环。

## 推荐阅读顺序

1. **Basics**：先建立组件 API 与样式基线认知。
2. **Forms & Inputs**：优先覆盖业务表单链路。
3. **Navigation + Feedback**：补齐核心交互和状态反馈。
4. **Data Display + Workflows**：进入复杂页面与业务流程搭建。

## 推荐使用流程

1. 在章节中找到目标组件并确认 API。
2. 使用 `npx formaui add <name>` 安装对应组件源码。
3. 在业务页面先做最小接入，再按场景扩展变体。
4. 联动 [Foundations](/docs/foundations) 检查 token、主题与无障碍约束。

## 适用角色

- 业务前端开发者
- 设计系统维护者
- 负责组件治理与交付规范的技术负责人

## 完成本章后的收益

- 组件选型有清晰路径，减少“重复造轮子”
- 业务页面组装速度更快，风格一致性更强
- 组件定制与后续升级具备稳定工程边界

如果你希望进一步提升页面搭建效率，建议继续阅读 [Blocks Overview](/docs/blocks/overview)。
