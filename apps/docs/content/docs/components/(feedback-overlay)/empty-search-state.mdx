# Empty Search State

`EmptySearchState` 用于在搜索或筛选无结果时提供恢复路径与下一步动作，帮助用户快速从“无匹配”回到可操作状态。

  <div className="flex min-h-[320px] items-center justify-center p-6">
    <EmptySearchStateInteractiveDemo />

</div>

```tsx title="components/demos/feedback-structure-demos.tsx"
import { EmptySearchState } from "@formaui/components";

export function NoResultPanel() {
  return (
    <EmptySearchState
      query="billing owner"
      clearLabel="Reset search"
      createLabel="Add teammate"
    />
  );
}
```

## API Reference

### Props

| Prop          | 类型         | 默认值                  | 说明                         |
| ------------- | ------------ | ----------------------- | ---------------------------- |
| `query`       | `string`     | `-`                     | 当前搜索词，用于拼接默认文案 |
| `title`       | `string`     | `"No matching results"` | 自定义标题                   |
| `description` | `string`     | `-`                     | 自定义描述文案               |
| `clearLabel`  | `string`     | `"Clear search"`        | 清空动作按钮文案             |
| `createLabel` | `string`     | `"Create item"`         | 创建动作按钮文案             |
| `onClear`     | `() => void` | `-`                     | 点击清空动作回调             |
| `onCreate`    | `() => void` | `-`                     | 点击创建动作回调             |
| `className`   | `string`     | `-`                     | 追加容器样式                 |

## 可访问性

- 容器使用 `aria-live="polite"`，可在状态变化时向读屏器播报更新。
- 两个动作按钮应使用明确动词，避免“确定/取消”等低语义文案。
- 建议保持一个“恢复查询”动作和一个“继续创建”动作，减少用户决策成本。
