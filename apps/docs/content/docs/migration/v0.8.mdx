# Migration: v0.7 to v0.8

FormaUI v0.8 rebuilds docs on Fumadocs-native content source (`content/docs`) and aligns IA to benchmark sections.

## Key Changes

- Canonical docs tree is now content-driven from `apps/docs/content/docs`.
- Primary route entry is `/docs/*`.
- Information architecture adopts top-level domains: Getting Started, Foundations, Components, Blocks, Templates, Guides, CLI, Registry, Migration, Release Notes, Resources.

## Upgrade Steps

1. Move or author docs pages under `content/docs/<domain>/*.mdx`.
2. Maintain page ordering through each domain `meta.json`.
3. Keep slugs stable and lowercase with kebab-case naming.
4. Run docs gates:

```bash
pnpm --filter docs test
pnpm --filter docs lint
pnpm --filter docs build
```

## Validation Checklist

- Sidebar renders all expected top-level domains.
- No orphan docs page outside `meta.json` ordering.
- Search and sitemap cover `/docs/*` routes.
