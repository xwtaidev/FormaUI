# Migration Guide: v0.4 -> v0.5

FormaUI v0.5 introduces a split-site architecture:

- `apps/web`: brand and growth site (landing, marketing, product, scenarios, showcase, blog, changelog)
- `apps/docs`: implementation documentation (installation, components, blocks, templates, CLI, registry, migration)

If you are upgrading from v0.4, follow the route mapping and rollout checklist below.

## Route Mapping (v0.4 -> v0.5)

| v0.4 Route (docs) | v0.5 Target | Notes |
| --- | --- | --- |
| `/landing` | `https://formaui.com/` | Landing now lives on web app |
| `/landing/architecture` | `https://formaui.com/product` | Product narrative moved to web |
| `/landing/reuse-blocks` | `https://formaui.com/showcase` | Showcase and reusable block story on web |
| `/` | `/` | Docs home remains docs-first |
| `/quick-start` | `/quick-start` | No route change |
| `/installation` | `/installation` | No route change |
| `/components/*` | `/components/*` | No route change |
| `/blocks/*` | `/blocks/*` | No route change |
| `/templates/*` | `/templates/*` | No route change |
| `/cli` and `/cli/commands` | `/cli` and `/cli/commands` | No route change |
| `/registry` | `/registry` | No route change |

## Upgrade Steps

1. Update bookmarks and internal links to point brand pages to `https://formaui.com`.
2. Keep all implementation links in docs pointed at `https://docs.formaui.com`.
3. Replace old landing links (`/landing`, `/landing/architecture`, `/landing/reuse-blocks`) with new web routes.
4. Add cross-site entry points:
   - Web header/footer: link to docs
   - Docs sidebar/top area: link to web
5. Validate routing with `pnpm --filter docs test` and `pnpm --filter docs build`.

## Validation Checklist

- Docs navigation contains `Migration v0.4 to v0.5`.
- Installation, Components, Blocks, Templates, CLI, and Registry routes are still reachable.
- Legacy marketing entry links are moved to web equivalents without dead links.
