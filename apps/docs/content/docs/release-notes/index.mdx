# Release Notes

本章节记录 FormaUI 的版本发布信息，采用统一结构，便于快速判断：

- 这个版本交付了什么
- 对现有项目有什么影响
- 升级前后应该做哪些验证

## 如何使用本章节

1. 先看目标版本的 **TL;DR** 与 **Highlights**。
2. 再看 **Upgrade Notes**，确认迁移动作。
3. 最后按 **Quality Gates** 执行验证命令。

- 新项目接入：优先看 `v0.8 -> v0.5`（文档与架构演进线）。
- 老项目升级：按当前版本向上逐版阅读 `Upgrade Notes`。
- 发布治理：关注每版 `Quality Gates` 与 `Known Limitations`。
