# ModelSelector

`ModelSelector` 用于在多个模型之间进行选择与比较，帮助团队在速度、成本和上下文窗口之间做工程取舍。

它内置 Provider 过滤、可选模型高亮和当前选择反馈，适合作为 AI 配置页或运行入口页的核心区块。

## 安装方式

### 通过 CLI 安装

```bash title="CLI"
npx formaui add model-selector
```

```bash title="Package Manager"
pnpm add @formaui/blocks
```

## 使用方式

```tsx title="Basic Usage"
import { ModelSelector } from "@formaui/blocks";

export function ModelRouteSection() {
  return <ModelSelector />;
}
```

## 案例

### 1. 自定义标题与说明

```tsx title="Custom Heading"
import { ModelSelector } from "@formaui/blocks";

export function WorkspaceModelPolicy() {
  return (
    <ModelSelector
      title="Workspace model policy"
      description="Choose a default model for new automations in this workspace."
    />
  );
}
```

### 2. 传入业务模型清单

```tsx title="Controlled Model List"
import { ModelSelector } from "@formaui/blocks";

const models = [
  {
    id: "gpt-5.5",
    name: "GPT-5.5",
    provider: "OpenAI",
    latencyMs: 420,
    costPerMillion: "$5.00",
    contextWindow: "256k",
    stability: "stable"
  },
  {
    id: "claude-sonnet-4",
    name: "Claude Sonnet 4",
    provider: "Anthropic",
    latencyMs: 510,
    costPerMillion: "$3.00",
    contextWindow: "200k",
    stability: "beta"
  }
] as const;

export function RoutedModels() {
  return <ModelSelector models={[...models]} />;
}
```

## API Reference

### Props

| Prop | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `models` | `DemoModelOption[]` | `demoModelOptions` | 模型候选列表 |
| `title` | `string` | `"Model selector"` | 区块标题 |
| `description` | `string` | `"Pick the best-fit model by speed, cost, and context limits."` | 区块说明 |

`DemoModelOption` 结构：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | `string` | 模型唯一标识 |
| `name` | `string` | 模型名称 |
| `provider` | `"OpenAI" \| "Anthropic" \| "Google"` | 模型提供方 |
| `latencyMs` | `number` | 延迟（毫秒） |
| `costPerMillion` | `string` | 每百万 token 成本 |
| `contextWindow` | `string` | 上下文窗口 |
| `stability` | `"stable" \| "beta" \| "preview"` | 稳定性标签 |

### 依赖组件

- `card`
- `badge`
- `button`
- `select`

## 响应式与可访问性

- 在 `md` 及以上断点自动切换为左右分栏，移动端保持单列。
- Provider 筛选基于可访问的 `Select` 组件，支持键盘导航。
- 模型选项使用 `button` 实现，可通过键盘聚焦与触发。
