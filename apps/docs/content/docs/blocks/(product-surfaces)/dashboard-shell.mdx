# DashboardShell

`DashboardShell` 是产品后台的基础布局区块，用于快速建立统一的控制台信息架构。

它默认组合了 `AppSidebar + PageHeader + MetricCard`，并通过 `children` 预留业务内容插槽。

## 安装方式

### 通过 CLI 安装

```bash title="CLI"
npx formaui add dashboard-shell
```

### 通过包依赖使用

```bash title="Package Manager"
pnpm add @formaui/blocks
```

## 使用方式

```tsx title="Basic Usage"
import { DashboardShell } from "@formaui/blocks";

export function ConsolePage() {
  return <DashboardShell />;
}
```

## 案例

### 1. 在内容区注入业务模块

```tsx title="With Content Slot"
import { DashboardShell, NotificationPanel, TokenUsageChart } from "@formaui/blocks";

export function OpsDashboard() {
  return (
    <DashboardShell>
      <div className="grid gap-4 lg:grid-cols-2">
        <TokenUsageChart />
        <NotificationPanel />
      </div>
    </DashboardShell>
  );
}
```

## API Reference

### Props

| Prop | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `children` | `ReactNode` | `undefined` | 注入主内容区的业务模块 |

### 依赖组件

- `app-sidebar`
- `page-header`
- `metric-card`

## 响应式与可访问性

- 在 `md` 及以上断点启用双栏布局（`16rem + 1fr`），移动端回落为单列。
- 结构化区域划分（导航 / 主内容 / 指标）有助于读屏和键盘导航。
