# Foundations Overview

Foundations 是 FormaUI 的工程基线章节，聚焦三个问题：

- 设计值如何统一表达（Design Tokens）
- 视觉主题如何稳定演进（Theme）
- 交互与内容如何满足可访问性要求（Accessibility）

如果你的团队正在把组件体系从“能用”升级为“可持续维护”，建议先完成本章节再进入大规模业务落地。

## 建议阅读顺序

1. **Design Tokens**：先统一设计语义与变量分层。
2. **Theme**：再建立 light/dark 与品牌主题策略。
3. **Accessibility**：最后把可访问性纳入发布门禁。

## 适用角色

- 设计系统维护者
- 前端基础架构负责人
- 需要统一多业务线 UI 规范的团队

## 完成本章后的收益

- 样式变更从“页面级修修补补”转为“语义变量级治理”
- 深色模式与品牌化定制成本显著下降
- 组件交付具备明确的可访问性验收标准
